    Hi,

    It seems that your app is on Rails 4.2.1. Rails later than 4.1 supports DATABASE_URL directly so the production: section of the database.yml file is not needed. Instead, can you try removing the line production: and below thatand setting the DATABASE_URL config var to point to the database you would like to use like the following?

    It seems that your app does not have its own Heroku Postgres. So, first, please pick up the app (here, I write <app-for-herokupostgres> that has the database for this app.

    Second, please find the add-on name of the database you would like to use. In the example below, the name is postgresql-fluffy-1234

    $ heroku addons -a <app-for-herokupostgres>
=== Resources for <app-for-herokupostgres>
Plan                         Name                    Price
---------------------------  ----------------------  -----
heroku-postgresql:hobby-dev  postgresql-fluffy-1234  free

=== Attachments for <app-for-herokupostgres>
Name                    Add-on                  Billing App
----------------------  ----------------------  ------------------------
DATABASE                postgresql-fluffy-1234  <app-for-herokupostgres>
HEROKU_POSTGRESQL_BLUE  postgresql-fluffy-1234  <app-for-herokupostgres>

    Third, please attach the database to your app as the primary database:

    $ heroku addons:attach postgresql-fluffy-1234 -a frozen-retreat-2087 --as DATABASE

    Finally, would you please try again pushing your code to Heroku to see how it goes?

    Moreover, you may want to roll the credentials for your databases as they have been exposed here.

    Thanks,
