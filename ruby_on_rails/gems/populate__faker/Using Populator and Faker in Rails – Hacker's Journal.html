Using Populator and Faker in Rails

Faker and Populator gems are used to fill your app database with dummy data. Add it to the Gemfile grouped under development( If you are testing only in development mode).

group :development do
  gem 'faker'
  gem 'populator'
end

Rake Task to populate data

We will start by writing our own rake task to populate data. Create a rake task under lib/tasks/[task-name].rake (Example lib/tasks/populate.rake)


Here is a sample rake task which is intended to create an Admin user.

namespace :db do
  desc "Create Admin user"
  task :populate => :environment do
  	admin = Admin.new(name: "Sudharshan", email: "user@example.com", admin: true)
  	admin.save
  end
end

To run the rake task

# rake [namespace]:[task]
rake db:populate

http://guides.rubyonrails.org/command_line.html#custom-rake-tasks

Faker and Populator

Before filling the database here is the documentation for Faker and Populator. Refer the documentation for methods and namespace that we are going to use in the rake task.

namespace :db do
  desc "Create 100 users with random names and descriptions"
  task :populate => :environment do
	require 'populator'
	require 'faker'
	  User.populate 100 do |user|
		user.name = Faker::Name.name
		user.email = Faker::Internet.email
		user.about = Faker::Lorem.sentence(word_count = 20, supplemental = false, random_words_to_add = 10)
		puts user.name
	  end
	  puts 'All done'
  end
end

First we add ‘populator’ and ‘faker’ to the take task. The ‘populator’ gem adds the method populate to all Active Record models. This is really useful while generating more than one record(Just pass the number of records to insert).

Faker has a number of namespaces (like Faker::Internet) each with a number of methods that return valid random content. For example Faker::Internet.email will return a valid email address that has been constructed from random content. The above rake task populates 100 users with random name, email and description. That’s it. Using Populator and Faker is pretty simple right? Watch the Railscast for more.
