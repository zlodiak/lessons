Django, django-sitetree, задача и решение, проекты, разработка, свободное ПО
Элементы навигации для Django: 3 в 1

В заметке речь пойдёт о новом приложении для Django — django-sitetree.

Приложение позволяет задавать структуру сайта в виде древа и предлагает три вида навигации по этому древу:

  1. Меню;
  2. Навигационная цепочка (хлебные крошки);
  3. Древо сайта (часто называемое «карта сайта»).

Для реализации первых двух элементов навигации можно легко отыскать отдельные Django-приложения (н.п. django-treemenus — меню, django-breadcrumbs — навигационная цепочка). Кстати сказать, вариантов навигационных цепочек чуть больше чем уйма и каждый из них, наверное, хорош по-своему.

Отличительной особенностью django-sitetree является как раз определение структуры сайта, в то время как различные типы навигации — суть средства перемещения в рамках структуры.

 

( Разработчикам о django-sitetree )

Настройка приложения

  1. Добавить 'sitetree' в список 'INSTALLED_APPS' файла конфигурации (обычно 'settings.py').
  2. Выполнить './manage.py syncdb' — в базу данных будут добавлены таблицы sitetree.
  3. Перейти к интерфейсу администрирования Django, задать древо сайта.
  4. Добавить '{% load sitetree %}' в начало шаблона.

Теперь в шаблоне вам доступны следующие тэги:

  • Тэг 'sitetree_menu' выводит меню:
    {% sitetree_menu from "mytree" include "trunk" %}

  • Тэг 'sitetree_breadcrumbs' выводит навигационную цепочку:
    {% sitetree_breadcrumbs from "mytree" %}

  • Тэг 'sitetree_tree' выводит древо сайта полностью:
    {% sitetree_tree from "mytree" %}


Подробнее о тэгах

  • Аргумент тэга определяющий древо (часть тэга в кавычках после слова 'from') может являться переменной шаблона.

  • Тэг 'sitetree_menu':
    {% sitetree_menu from "mytree" include "trunk,topmenu" %}
    Выведет элементы являющиеся дочерними по отношению к элементам дерев с псевдонимами 'trunk' и 'topmenu'.

    Псевдонимы задаются элементам через интерфейс администрирования. Следующие псевдонимы зарезервированы:

    • 'trunk' — элементы без родителей (корневые элементы)
    • 'this-children' — элементы дочерние для элемента, распознанного для данной страницы как текущий
    • 'this-siblings' — элементы имеющие одного родителя с элементом, распознанным для данной страницы как текущий (включая сам текущий элемент)
    К элементам древа можно обращаться не только по псевдонимам, но и по ID.
  • Необязательный для тэгов аргумент 'template' может использоваться с любым из упомянутых выше тэгов:
    {% sitetree_menu from "mytree" include "trunk,topmenu" template "mytrees/mymenu.html" %}
    {% sitetree_breadcrumbs from "mytree" template "mytrees/mybreadcrumbs.html" %}


Замещение встроенных шаблонов

Для того, чтобы изменить вид элементов навигации, потребуется заместить встроенные шаблоны:

  1. Перейти в директорию с установленным sitetree;
  2. Переместиться ниже в 'templates/sitetree';
  3. Здесь обретаются файлы встроенных шаблонов:
    • breadcrumbs.html
    • menu.html
    • tree.html
  4. Скопируйте те из них, которые хотите изменить в директорию шаблонов вашего проекта. После чего можете делать с ними всё, что угодно.
  5. Обратитесь к разделу 'Заметки о «внутренних» тэгах sitetree', расположенному ниже, за описанием некоторых «внутренних» тэгов, использующихся в шаблонах sitetree.


Заметки о «внутренних» тэгах sitetree

SiteTree предоставляет пару дополнительных тэгов, с которыми вам придётся столкнуться, в случае замещения встроенных шаблонов.

  • Тэг 'sitetree_children':
    {% sitetree_children of someitem for menu template "sitetree/mychildren.html" %}
    Реализует рекурсивный проход вниз по дереву.
    Используется для вывода дочерних элементов заданного элемента дерева 'someitem', используя шаблон "sitetree/mychildren.html" для типа навигации 'menu'.
    В общем случае аргумент шаблона должен указывать путь до самого шаблона.
    Допустимы следующие типы навигации: 1) menu; 2) sitetree.

  • Тэг 'sitetree_url' :
    {% sitetree_url for someitem params %}
    Вычисляет URL, либо URL-шаблон элемента древа.
    Тэг очень похож на встроенный в Django тэг 'url'. Различие заключается в том, что после слова 'for' должен следовать объект элемента древа.
    Да, после объявления объекта, можно перечислить параметры (как в тэге 'url').

«trunk» обнаруживается все элементы без родителей и на их основе строит полноценное меню.

Если нужно, чтобы меню не включало потомков, следует использовать подправленный шаблон menu.html. Например, назвать его my_menu.html и отрисовать при помощи

{% sitetree_menu from "main_menu" include "trunk" template "my_sitetree/my_menu.html" %}

В шаблоне остаётся убрать строки

        {% if item.has_children %}
            {% sitetree_children of item for menu template "sitetree/menu.html" %}
        {% endif %}
