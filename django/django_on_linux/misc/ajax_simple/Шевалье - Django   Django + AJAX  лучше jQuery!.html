Django | Django + AJAX: лучше jQuery!

  • Nov. 15th, 2011 at 1:58 PM

Простой пример применения технологии AJAX в сочетании с jQuery и Django приведен в этой статье:
AJAX и Django + jQuery

Однако я немного видоизменил код js-скрипта, чтобы он не запускался каждый раз при загрузке страницы, а выполнялся только по моей команде. Для этого на тестовую страницу я добавил следующий элемент:

<p><a id="test">Hello</a></p>

Далее, в верхней части страницы в положенном месте я также добавил следующий код:

<script type="text/javascript" src="{{MEDIA_URL}}js/jquery-1.7.js"></script>
<script type="text/javascript">
   $(function() {
      $("#test").click(function() {
         $.get("/xhr_test/", function(data) {
            alert(data);
         });
      });
    });
</script>

Переменная MEDIA_URL - это локальная ссылка '/media/'. При загрузке страницы браузер получает src="/media/js/jquery-1.7.js".

В файле views.py:

def xhr_test(request):
    if request.is_ajax():
        message = "Hello AJAX!"
    else:
        message = "Hello"
    return HttpResponse(message)


И наконец, urls.py:

urlpatterns = patterns('',
    (r'^xhr_test/$', xhr_test),
)


В общем и целом, практически одно и то же, только приветствие 'Hello AJAX!' выскакивает при клике по строке 'Hello'. Этот эффект достигается благодаря использованию в jquery-коде функции click():
http://api.jquery.com/click/

Что же касается функции $get(), то ее описание можно прочитать в этой части документации:
http://api.jquery.com/jQuery.get/
