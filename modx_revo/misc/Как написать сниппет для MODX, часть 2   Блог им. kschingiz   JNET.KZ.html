Как написать сниппет для MODX, часть 2

Блог им. kschingiz / 25 июля 2013, 17:30
Всем привет, я представляю вам вторую часть статьи из цикла “Как написать сниппет для MODX”, и сегодня мы познакомимся с чанк процессингом, операциями над ресурсами, программного подключения внешних файлов и тд.
Очень надеюсь, что статья будет полезной для Вас.

Операции над ресурсами


Есть класс modX, в котором определены различные методы, в том числе и работа с ресурсами, пользователями, чанками и тд.
Полный список доступных методов можете посмотреть здесь rtfm.modx.com/display/revolution20/modX
Так как modX наследует класс xPDO, вы можете пользоваться и его методами:
rtfm.modx.com/display/XPDO10/xPDO
Внутри сниппета объектом класса modX является $modx, а xPDO — $xpdo.
Для того, что получить что-то, мы должны воспользоваться методом:
getObject(TYPE, $id);

для ресурсов вместо TYPE, пишем modResource
для юзверей – modUser
Второй параметр, ясень пень, — идентификатор ресурса или пользователя.
Давайте возьмем ресурс с ID = 5 и вернем его контент:
$res = $modx->getObject('modResource', 5);
return $res->content;

во второй строке мы возвращаем свойство content, в котором хранится контент ресурса.
Ниже я привожу таблицу свойств modResource:

Сделаем такой бесполезный сниппет, в которую мы передаем ID документа, а сниппет выводит его содержимое на страницу, я назвал его showResource:
$id = isset($id) ? $id : 1;
$res = $modx->getObject('modResource', $id);
return $res->content;

и вызываем где-нибудь:
[[!showResource? &id=`6`]]

Результат:

Также вы можете воспользоваться любым свойством из таблицы, но если вам необходимо возвращать свойства и плюс еще и разметку, ни в коем случае не пишите, что-то типа такого:
return '<h1>'. $res->pagetitle . '</h1> <p>' . $res->content . '</p>';

потому что, во первых, в сниппете должен быть только РНР код, а во вторых, задачу эту можно решить более лаконично, прочитав эту статью до конца.

Чанк Процессинг


Что такое чанк?
Если сниппет — кусок РНР кода, то чанк это кусок HTML кода. В чанках мы можем оставить плейсхолдеры, которые в дальнейшем будут спроцессированы сниппетом.
К примеру, у нас имеется чанк tpl_res с такой разметкой:
<h2> [[+title]] </h2>
<p> [[+content]] </p>

title и content, в данном случае, являются плейсхолдерами.
Передав данный чанк в сниппет, мы можем указать сниппету, что вместо плейсхолдеров надо вывести соответствующие данные. Это и есть чанк процессинг.
Возвращаемся к нашему сниппету, давайте сделаем так, чтобы он работал с чанками.

$id = isset($id) ? $id : 1;
$res = $modx->getObject('modResource', $id);

$placeholders = array(
    'title' => $res->pagetitle,  // Вместо плейсходера title выводим название страницы
    'content' => $res->content   // Вместо плейсходера content выводим содержимое
);
$output = $modx->getChunk('tpl_res', $placeholders);

return $output;


В этом сниппете мы объявляем ассоциативный массив ключом, которого являются наши плейсхолдеры, а значения – нужные данные. После, мы берем чанк tpl_res и передаем в него наш массив, MODX автоматически заменят все плейсхолдеры на нужные данные из этого массива.
Вызовем наш сниппет:
[[!showResource? &id=`8`]]

и видим следующее:

Обратите внимание на разметку, она формируется в соответствии с нашим чанком. Чанк в данный момент служитшаблоном вывода для сниппета, будет лучше, если такие чанки в названии будут иметь префикс tpl_, чтобы легче было опрeделить чанк это шаблон для сниппета или нет.
Добавим такой функционал – в данный сниппет мы передаем, название чанка, которую будем процессировать, а если не передаем, то используем дефолтный чанк (tpl_res).
Создаем чанк tpl_content:
<h2>[[+title]]</h2>
<p>[[+description]]</p>
<p>[[+content]]</p>

Как видим, в чанке выше мы добавили плейсхолдер для описания ресурса, модифицируем сниппет:
$tpl = isset( $tpl ) ? $tpl : 'tpl_res'; // шаблон вывода
$id = isset($id) ? $id : 1; 
$res = $modx->getObject('modResource', $id);

$placeholders = array(
    'title' => $res->pagetitle,
    'description' => $res->description,
    'content' => $res->content
);
$output = $modx->getChunk($tpl, $placeholders);

return $output;

Теперь у нас есть несколько вариантов вызова нашего сниппета:
[[!showResource]]  - выводит ресурс с ИД = 1
[[!showResource? &id=`8`]] - выводит ресурс с ИД = 8, используя дефолтный шаблон вывода  tpl_res
[[!showResource? &tpl=`tpl_content` &id=`8`]] - выводит ресурс с ИД = 8, используя шаблон  tpl_content

Программное подключение стилей и скриптов


При использовании сниппетов часто возникает необходимость подключения внешних фалов, будь то css, js. И будет удобней, если нам не придется самим лазить в шаблон и подключать файлы, а чтобы сниппет программно подключал их.
Усложним наш сниппет, сделаем так чтобы он подключал стили, и через параметр useStyle, можно было решать какой именно стиль нам надо подключить.
Для начала создаем дефолтный стиль defalut.css сохраняем в папке assets/styles/:
h2{
    margin-bottom: 5px;
    color: rgb(29, 87, 142);
    font-size: 18px;
    font-family: Georgia;
}
p{
    font-size: 22px;
    margin-top: 2px;
    margin-bottom: 10px;
    color: rgb(202, 65, 116);
}

Стили эти спионерил с jnet, надеюсь разрабы не обидятся. Возвращаемся в наш сниппет и дописываем эти две строки:
$useStyle = isset ($useStyle) ? $useStyle : 'assets/styles/default.css'; // В начало сниппета
$modx->regClientCSS($useStyle); // в конец, но только до return

Смотрим на вторую строку, где мы методом regClientCSS подключаем нужный нам стиль.
Результат:

Можете создать стиль и подключить его. Например вот таким вызовом:
[[!showResource? &useStyle=`assets/styles/YOUR_STYLE.css` ]]

Похожие методы:
– regClientScript  - подключаем JS файлы
– regClientHTMLBlock – HTML файл

Как видим ничего сложного нет.
