Попробую объяснить. Я использую Python 3.1, но постараюсь дать комментарии и для ветки 2.x.

Начнём со словарей.
Значения словаря хранятся в неотсортированном порядке.

Сортировка словаря по ключам:

>>> a = {'a':1,'d':4,'c':3,'b':2}
>>> a
{'a': 1, 'c': 3, 'b': 2, 'd': 4}
>>> for key in sorted(a):
	print(key +'=>'+ str(a[key]));

a=>1
b=>2
c=>3
d=>4
>>>

Или так.

>>> a = {'a':1,'d':4,'c':3,'b':2}
>>> b = a.keys()
>>> b = list(b)
>>> b.sort()
>>> for key in b:
    print(key +'=>'+ str(a[key]));

a=>1
b=>2
c=>3
d=>4
>>>

Если используется Python 2.x то не нужно преоразоывать итератор в список функцией list, т.к. метод a.keys() и так возвращает список, а не итератор как в Python 3.x. И функцию print() нужно заменить на инструкцию print.

Сортировка словаря по значениям:

>>> a = {'a':1,'d':4,'c':3,'b':2}
>>> a
{'a': 1, 'c': 3, 'b': 2, 'd': 4}
>>> b = list(a.items())
>>> b
[('a', 1), ('c', 3), ('b', 2), ('d', 4)]
>>> b.sort(key=lambda item: item[1])
>>> for item in b:
    print(item[0] +'=>'+ str(item[1]))

a=>1
b=>2
c=>3
d=>4
>>>

В Python 2.x так же можно обойтись без преобразования в list. Так же до версии 2.6 метод sort принимал функцию сравнения cmp. Тоесть она должна выглядить как-то так:

def cmp(x, y):
    return 1 if x[1] > y[1] else -1 if x[1] < y[1] else 0

b.sort(cmp)

Либо так b.sort(lambda x, y: return 1 if x[1] > y[1] else -1 if x[1] < y[1] else 0); Хотя в этом способе я не уверен.

Сортировка списка по значениям:

Тут нету ничего сложного

>>> a = [2,5,1,6,3,4]
>>> a
[2, 5, 1, 6, 3, 4]
>>> a.sort()
>>> a
[1, 2, 3, 4, 5, 6]
>>>
